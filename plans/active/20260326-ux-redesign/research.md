# Eazy HWPX UX 리디자인 리서치 (2026-03-26)

## 트랙 A: UX 온보딩
- **3단계, 3분 이내** 첫 결과물 (7단계 투어 완료율 16%)
- **"샘플로 체험하기"** 버튼 필수 — 파일 없어도 바로 체험
- **전체 탭 노출 + AI 매핑에 "추천" 뱃지** (독립 기능은 숨기지 않음)
- **5초 테스트** 통과: 헤드라인 1줄로 가치 전달
- **Empty State**: "없습니다" 금지 → 샘플 + CTA

## 트랙 B: 콘텐츠/카피
- **PAS**: 문제("양식에 하나하나 타이핑") → 증폭("매달 반복") → 해결("올리면 자동")
- **토스 원칙**: 쉽게, 간결, 빈 문장 제거, 기술 용어 금지
- **원라이너**: "[숫자]+[결과]" = "10분 만에 양식 30장 완성"
- **Feature→Benefit**: "AI 매핑" → "양식 칸을 자동 인식"
- **히어로 3요소**: 헤드라인(What+Why) + 서브(How+Who) + CTA(행동 동사)
- **Benefit을 위에, Feature는 아래로**

## 트랙 C: 시각 레이아웃
- **Z-Pattern**: 좌상(제목) → 우상(CTA) → 좌하(사이드바) → 우하(작업)
- **상태 전환**: 미업로드 시 히어로 크게 → 업로드 후 히어로 축소
- **탭 7개**: 아이콘+짧은 라벨, 스크롤 가능, 그룹핑 구분자
- **간격 8px 단위**: 8, 16, 24, 32, 48
- **카드**: 히어로용 / **인라인**: 작업 중 가이드 / **리스트**: 파일 목록

## 트랙 D: 경쟁 도구
- **iLovePDF**: 카드 그리드로 25+도구 한눈에, 제로 마찰(로그인 없이 사용)
- **Smallpdf**: 미니멀, 드래그앤드롭, 1순위 CTA 명확
- **PandaDoc**: 히어로에 파일 업로드 영역이 메인 CTA
- **DocuSign**: 3단계 체크리스트 + 프로그레스 바
- **한컴독스**: 파일 중심 UI (기능이 아닌 파일 먼저 선택)
- **정부24 교훈**: 기능 숨기면 실패, 인증 장벽은 적

## 기존 패턴 (필수 준수)
- "디자인 변경 전 비즈니스/마케팅 리서치 선행" (critical) ← 이번에 수행
- "첫 페이지에서 스크롤만으로 모든 핵심 정보" (critical)
- "예쁜 것보다 뭘 할 수 있는지, 소비자 이득"
- "과장 금지, 팩트만"
- "기술 용어 금지"
