# Eazy HWPX UX 리디자인 실행 계획

## 현재 문제 (테스터 피드백 + 리서치 기반)
1. 첫 방문자가 "뭘 하는 도구인지" 30초 안에 파악 불가
2. 7개 기능이 있는데 1개(AI 매핑)만 설명됨
3. "왜 써야 하는지" 이점이 안 보임
4. 시각적 계층/정렬 미흡
5. 파일 없이 체험할 방법 없음

## 설계 원칙 (리서치 기반)
- PAS: 문제 → 증폭 → 해결
- 5초 테스트 통과하는 헤드라인
- 3단계 3분 이내 첫 결과물
- Z-Pattern 시선 흐름
- Feature→Benefit 변환
- 토스 라이팅 (쉽게, 간결, 빈 문장 제거)
- iLovePDF식 카드 그리드 (기능 발견성)

## 실행 계획

### Phase 1: 페이지 구조 재설계 (page.tsx)

```
[헤더] Eazy HWPX (좌측 정렬)

[히어로 — Above the fold]
  좌측:
    헤드라인: "아직도 양식에 하나하나 입력하세요?"  (Problem)
    서브: "양식 파일을 올리면, AI가 내용을 자동으로 채워줍니다" (Solution)
    CTA 2개:
      [Primary] "내 파일로 시작하기" → 파일 업로드 영역으로 스크롤
      [Secondary] "샘플로 체험하기" → 샘플 HWPX 자동 로드
  우측:
    3단계 흐름 (세로 배치)
      1. 양식 업로드 — HWP / HWPX / DOCX
      2. 내용 채우기 — 붙여넣기, 엑셀, AI 작성
      3. 완성 다운로드 — HWP / HWPX / DOCX

[기능 카드 그리드 — iLovePDF 스타일]
  7개 기능을 아이콘 + 제목 + Benefit 한 줄로
  AI 매핑 카드에 "추천" 뱃지
  카드 클릭 → 해당 탭으로 이동 + 양식 넣기 강조

[도구 영역 — 작업 공간]
  왼쪽: 양식 넣기 (사이드바)
  오른쪽: 탭 + 콘텐츠 (현재 유지)

[데이터 안내]
  토글 유지 (현재대로)
```

### Phase 2: 카피라이팅

| 위치 | 현재 | 변경 |
|------|------|------|
| 헤드라인 | "양식은 그대로, 내용만 바꿔드립니다" | "아직도 양식에 하나하나 입력하세요?" |
| 서브 | "기존 양식에 다른 내용을..." | "양식 파일만 올리면, AI가 알아서 채워줍니다" |
| CTA | "양식 분석" | "내 파일로 시작하기" / "샘플로 체험하기" |
| 기능 카드 | 탭 라벨만 | Benefit 한줄 추가 |

### Phase 3: 기능 카드 Benefit 변환

| 기능 | Feature | Benefit |
|------|---------|---------|
| AI 매핑 | AI 자동 매핑 | 주제만 알려주면 문서를 대신 써줍니다 |
| 엑셀→문서 | 엑셀 데이터 일괄 생성 | 엑셀 100행이면 문서 100개가 한번에 |
| 문서→엑셀 | 텍스트 추출 | 문서 안의 모든 글자를 엑셀로 정리 |
| 정기문서 | 날짜 치환 반복 생성 | 매달 보고서를 12개월치 한번에 |
| 도장 | 도장 이미지 삽입 | (인) 자리에 도장을 자동으로 찍어줍니다 |
| 합치기 | 문서 병합 | 여러 파일을 하나로 합쳐줍니다 |
| 엑셀 채우기 | 엑셀 셀 자동 입력 | 엑셀 양식의 빈칸을 자동으로 채웁니다 |
